# 管理后台使用说明

FWP 管理后台分为 4 个功能区域：**概览**、**管理**、**监控**、**系统**。

## 仪表盘（概览）

仪表盘提供系统运行状态的全局视图，包括：

- **平台统计**: 总平台数、活跃平台数、健康状态分布
- **Key 统计**: 总 Key 数、活跃 Key 数
- **请求统计**: 总请求数、错误请求数、成功率
- **Token 统计**: 总 Token 消耗量
- **性能指标**: 平均首 Token 时间（TTFT）、平均请求耗时
- **趋势图表**: 各指标的迷你趋势图
- **最近事件**: 最新的系统事件列表

仪表盘默认每 30 秒自动刷新，也可手动点击刷新按钮。支持网格视图和详情视图两种模式。

## 平台管理

平台管理用于配置和管理上游 AI 服务提供商。

### 添加平台

点击「添加平台」，填写以下信息：

| 字段 | 说明 | 必填 |
|------|------|------|
| 平台名称 | 自定义标识名称 | 是 |
| Base URL | 平台 API 地址（如 `https://api.openai.com`） | 是 |
| API Key | 平台的认证密钥 | 是 |
| 附加密钥 | 额外的密钥列表（JSON 数组格式），与主密钥轮询使用 | 否 |
| 平台类型 | `openai` / `azure` / `custom` | 是 |
| 优先级 | 数值越大优先级越高 | 否（默认 0） |
| 权重 | 负载均衡权重，数值越大分配越多 | 否（默认 1） |
| RPM 限制 | 每分钟最大请求数 | 否 |
| TPM 限制 | 每分钟最大 Token 数 | 否 |
| 透传请求头 | 需要转发给上游的下游请求头白名单（JSON 数组） | 否 |

### 健康检查与熔断

FWP 内置自动健康检查机制：

- **healthy**: 正常状态，正常接收请求
- **degraded**: 性能下降状态，降低分配频率
- **down**: 故障状态，触发熔断，不分配请求

**熔断规则**:
- 连续失败 5 次后触发熔断，状态变为 `down`
- 熔断后进入 60 秒冷却期
- 冷却期结束后自动进入 `half-open` 状态，发送探测请求
- 探测成功则恢复为 `healthy`，失败则重新熔断

### 密钥轮询

当平台配置了多个密钥（主密钥 + 附加密钥）时，FWP 采用 Round-Robin 轮询策略，均匀分配请求到各个密钥，避免单个密钥调用量过高。

附加密钥格式：

```json
[
  {"name": "密钥1", "key": "sk-xxx"},
  {"name": "密钥2", "key": "sk-yyy"}
]
```

## API Key 管理

API Key 是客户端访问 FWP 的凭证。

### 创建 API Key

点击「添加 Key」，填写以下信息：

| 字段 | 说明 | 必填 |
|------|------|------|
| 名称 | 自定义标识名称 | 是 |
| 套餐 | 绑定预定义的套餐模板（可选，覆盖下方自定义值） | 否 |
| Token 额度 | 总 Token 使用上限 | 否（默认 0） |
| 调用次数限制 | 总调用次数上限 | 否（默认 0） |
| RPM 限制 | 每分钟请求数限制 | 否（默认 0） |
| TPM 限制 | 每分钟 Token 数限制 | 否（默认 0） |
| 重置周期 | `monthly` / `daily` / `never` | 否（默认 monthly） |
| 过期时间 | 到期后 Key 自动禁用 | 否 |

### 配额规则

- 额度值为 `0` 表示不设限制
- 当 `planId` 未指定时，使用自定义的额度值
- 当 `planId` 已指定时，套餐模板中的值优先（除非自定义值明确设置）

### 套餐模板

套餐模板预定义了配额标准，便于快速创建 Key。在系统设置中管理套餐模板。

### 自动重置

根据 `resetPeriod` 设置自动重置已用 Token 数：
- `monthly` — 每月 1 日凌晨重置
- `daily` — 每天凌晨重置
- `never` — 永不重置

### Key 状态

- **启用**: 正常接受请求
- **禁用**: 拒绝所有请求（返回 403）
- **过期**: 到达过期时间后自动禁用
- **超限**: Token 用量或调用次数达到上限后拒绝请求

## 模型映射

模型映射允许将客户端请求的模型名映射到实际调用的模型名。

### 配置映射

| 字段 | 说明 |
|------|------|
| 别名 (Alias) | 客户端请求的模型名 |
| 目标模型 (Target) | 实际转发到上游的模型名 |
| 平台 | 限定在特定平台生效（留空则由路由引擎自动选择） |

### 使用示例

| 别名 | 目标模型 | 平台 | 效果 |
|------|----------|------|------|
| `gpt-4` | `gpt-4o` | — | 所有平台生效 |
| `claude-3` | `claude-3-5-sonnet` | Anthropic 平台 | 仅 Anthropic 生效 |

### 使用场景

1. **模型升级**: 旧模型名映射到新模型（如 `gpt-4` -> `gpt-4o`）
2. **成本优化**: 将昂贵模型映射到性价比更高的模型
3. **兼容适配**: 客户端使用固定模型名，后端灵活切换

### 自动模型（Auto Model）

自动模型是 FWP 的高级路由功能：

1. 系统每 10 分钟自动发现各平台支持的模型列表
2. 在「自动模型」页面可以查看各平台的模型发现结果
3. 可以选择特定模型加入自动路由池
4. 系统自动生成一个自动模型 ID，客户端使用该 ID 时，FWP 会自动选择最佳平台和模型
5. 自动模型请求失败时会临时冻结 3 分钟，避免反复失败

## 代理管理

### 代理池

代理池是一组 HTTP 代理的分组管理单元。

| 字段 | 说明 |
|------|------|
| 名称 | 池的唯一标识名称 |
| 启用 | 是否启用该池 |

### 添加代理

在「代理管理」页面添加具体代理：

| 字段 | 说明 |
|------|------|
| 地址 | 代理地址，支持 HTTP 和 SOCKS5 |
| 所属池 | 绑定到哪个代理池 |
| 启用 | 是否启用该代理 |

地址格式：
- HTTP: `http://user:pass@host:port`
- SOCKS5: `socks5://user:pass@host:port`

### 代理批量导入

支持通过文本格式批量导入代理地址，每行一个地址。

### 代理健康检查

FWP 后台自动检测代理健康状态：

| 状态 | 说明 | 路由行为 |
|------|------|----------|
| healthy | 正常可用 | 正常使用 |
| degraded | 性能下降 | 随机 50% 跳过 |
| down | 不可用 | 完全跳过 |

**封禁递增机制**:
- 首次封禁：冷却 15 分钟
- 第 2 次封禁：冷却 5 小时
- 第 3 次及以上：冷却 24 小时

### 代理路由

请求转发时的代理选择逻辑：
1. 只选择 `healthy` 或 `degraded` 状态的代理
2. 并发感知轮转：同一平台的不同 Key 优先分配到不同代理
3. 代理失败自动重试（最多 2 次不同代理）
4. 所有代理均失败后回退为直连

## 用量监控

### 用量统计

用量统计页面提供多维度的数据分析：

**趋势图表**:
- 支持按月/按周/按天查看请求量和 Token 用量趋势
- 支持单日小时级粒度展示

**Key 用量标签页**:
- 按 API Key 维度查看请求量、Token 用量、平均 TTFT

**平台用量标签页**:
- 按平台维度查看请求分布、Token 消耗、性能对比

### 请求日志

记录每一次 API 请求的详细信息：

| 字段 | 说明 |
|------|------|
| API Key | 使用的 Key 名称 |
| 平台 | 转发到的平台名称 |
| 模型 | 请求的模型名 |
| 状态码 | HTTP 响应状态码 |
| Token 用量 | prompt + completion 总 Token 数 |
| TTFT | 首个 Token 时间（流式响应有效） |
| 耗时 | 请求总耗时 |
| 代理 | 使用的代理 ID（直连则为空） |
| 错误信息 | 失败时的错误详情 |

### 日志归档

- 超过 30 天的详细请求日志自动聚合为每日统计数据
- 聚合维度：日期 + API Key + 模型
- 支持手动触发日志归档

### 审计日志

记录所有管理员操作，包括：
- 登录/登出
- 创建/更新/删除平台
- 创建/更新/删除 API Key
- 修改系统配置
- 数据导入/导出

每条审计日志包含操作者、操作类型、操作详情、客户端 IP。

### 系统事件

记录系统级事件，按严重程度分为四级：
- **info**: 系统初始化、配置变更等
- **warning**: 平台性能下降、Key 额度接近上限等
- **error**: 平台故障、请求异常等
- **critical**: 所有平台不可用等严重问题

## 数据管理

### 数据导出

支持三种导出类型：
- **系统配置**: 平台、API Key、模型映射、代理、套餐等配置数据
- **业务数据**: 请求日志、每日统计、审计日志、系统事件
- **全部数据**: 以上所有内容

导出文件为 JSON 格式。

### 数据导入

支持导入之前导出的 JSON 文件，用于：
- 迁移到新服务器
- 恢复备份数据
- 同步配置

导入时会显示每个数据类型的导入数量和跳过数量。

## 系统设置

系统设置页面提供：
- 系统运行状态概览（数据库连接、平台数量、Key 数量）
- 管理员密码修改功能
